LeetCode 7: Reverse Integer

Medium #7 View on LeetCode
Java Math

Problem Statement

Given a signed 32-bit integer x, return x with its digits reversed. If reversing x causes the value to go outside the signed 32-bit integer range [-2^31, 2^31 - 1], then return 0.

Approach

The solution reverses the integer by repeatedly extracting the last digit using the modulo operator %, appending it to the result, and dividing the input by 10 to remove the last digit. It handles negative numbers by reversing the positive equivalent and negating the final result. It also checks for overflow by comparing the current result with Integer.MAX_VALUE / 10 before appending each digit.

Code Solution (Java)

Explanation

  1. Initialize res to store the reversed integer, remainder for the last digit, and isNegative flag to handle negative inputs.
  2. If x is negative, negate it and set isNegative to true.
  3. Enter a loop that continues as long as x > 0:
    • Check if appending the next digit will cause overflow by comparing res with Integer.MAX_VALUE / 10. If so, return 0.
    • Multiply res by 10 to shift the existing digits left.
    • Extract the last digit of x using x % 10 and store it in remainder.
    • Add remainder to res.
    • Divide x by 10 to remove the last digit.
  4. If isNegative is true, negate res.
  5. Return the final reversed integer res.

Complexity Analysis

Time Complexity: O(log x), where x is the input integer. The number of iterations depends on the number of digits in x, which is logarithmic.

Space Complexity: O(1), as the solution uses only a constant amount of extra space.
